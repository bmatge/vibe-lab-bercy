# 5. Calendrier — première vague (12 semaines)
Le plan se déploie en trois temps : d’abord sécuriser le soutien hiérarchique, ensuite générer la demande métier, enfin communiquer largement.

# 6. La règle d’or

La crédibilité du Lab repose sur les résultats, pas sur les promesses. Chaque communication doit s’appuyer sur un outil existant, un chiffre vérifiable, un témoignage réel. Le jour où le Lab communique sur un projet qui n’est pas encore livré, il perd sa force principale : la preuve par l’exemple.

| Le meilleur projet du monde échoue s’il n’est pas connu. Aujourd’hui le Vibe Lab a 8 outils en production, 663 K€ de valeur démontrée — et une poignée de personnes au courant de son existence.

Ce plan de communication vise à transformer la visibilité du Lab pour générer la demande : que chaque direction sache ce que le Lab peut faire pour elle, et que les décideurs disposent des arguments pour le soutenir. |
| --- |

| Symptôme | Cause |
| --- | --- |
| Les directions ne proposent pas de projets | Elles ne savent pas que le Lab existe ou ce qu’il peut faire pour elles |
| Le Lab choisit seul ses sujets | Risque de décalage avec les vrais besoins métier |
| Les décideurs ne peuvent pas le défendre | Ils n’ont pas les arguments face à un arbitrage budgétaire |
| BercyHub ne connaît pas les limites | Risque de confusion de périmètre par méconnaissance |
| Les agents ne savent pas qu’ils peuvent demander | Des problèmes restent non résolus alors que la solution est à portée |

| L’objectif n’est pas de « faire de la com’ ». C’est de générer la demande. Un Lab sans demande entrante est un Lab qui se parle à lui-même. |

| A. Direction générale / Secrétariat général
Qui : SG, directeurs, cabinet, n+2 du SNUM
Ce qu’ils redoutent : Que ça dérape (sécurité, données, image), que ça fasse doublon avec BercyHub, que ça coûte sans résultat visible
Le message : « Le Vibe Lab a produit 663 K€ de valeur en 6 mois, avec 1 agent et ~1 000 €/mois. Zéro donnée sensible, zéro interférence BercyHub. C’est le meilleur ratio coût/impact du ministère. »
Le format qui convainc : Note de synthèse 2 pages (déjà produite), 10 min en comité de direction avec démo live |
| --- |

| B. Directions métier (DRH, DAF, DIRCOM, DSI, Immobilier)
Qui : Directeurs et sous-directeurs, chefs de bureau, référents numériques
Ce qu’ils redoutent : Que ça leur crée du travail supplémentaire, que ça ne réponde pas à leur vrai problème, que le prototype disparaisse après la démo
Le message : « Dites-nous votre problème, on vous montre un prototype en 2 semaines. Si ça ne sert pas, on arrête. Si ça sert, on pérennise. Vous ne risquez rien. »
Le format qui convainc : Démo personnalisée sur un vrai fichier/problème de la direction, en 30 min |
| --- |

| C. BercyHub
Qui : L’équipe BercyHub, son responsable, sa tutelle
Ce qu’ils redoutent : Que le Vibe Lab empiète sur leur périmètre, qu’il décrédibilise leur approche souveraine, qu’il crée de la confusion
Le message : « Le Vibe Lab est votre éclaireur. On prototyppe sur données publiques, on valide le concept, vous industrialisez sur données sensibles. On vous fait gagner 6 mois et 100 K€ de cadrage. »
Le format qui convainc : Café informel d’abord, puis point trimestriel structuré |
| --- |

| D. Agents et encadrement intermédiaire
Qui : Chefs de bureau, gestionnaires, chargés de mission, assistants
Ce qu’ils redoutent : Que l’IA les remplace, que ça change leurs habitudes sans les consulter, que ce soit un « truc de geeks »
Le message : « Le Vibe Lab fabrique des outils pour vous, pas à votre place. Vous avez un fichier Excel pénible ? Un process répétitif ? Dites-le nous, on simplifie. »
Le format qui convainc : Article intranet avec avant/après concret, témoignage d’un collègue |
| --- |

| E. Communauté tech interne et interministérielle
Qui : Développeurs SNUM, équipes DINUM, beta.gouv, réseau des labs d’innovation publique
Ce qu’ils redoutent : Que ce soit du bricolage non professionnel, que le code ne soit pas à la hauteur
Le message : « 238K LOC, 1 800+ tests, CI/CD, DSFR natif, open source sur GitHub. Le vibe coding n’est pas du no-code — c’est du code professionnel accéléré par l’IA. »
Le format qui convainc : README exemplaire sur GitHub, talk en meetup interne, article technique sur le blog SNUM |
| --- |

| F. Organisations syndicales
Qui : Représentants du personnel, CHSCT, CTM
Ce qu’ils redoutent : Que l’IA supprime des postes, que les conditions de travail changent sans concertation
Le message : « Le Vibe Lab améliore les conditions de travail des agents en supprimant les tâches répétitives. Aucun poste n’est menacé — au contraire, les agents gagnent du temps pour leur cœur de métier. »
Le format qui convainc : Information en CTM si nécessaire, avec exemples concrets de simplification du quotidien |

| Objection | Réponse |
| --- | --- |
| « C’est un gadget, ça ne tiendra pas » | 8 outils en production depuis 6 mois, 663 K€ de valeur mesurée, code open source sur GitHub. Si c’était un gadget, les chiffres ne seraient pas là. |
| « Ça fait doublon avec BercyHub » | Périmètres strictement séparés : données publiques vs. données sensibles, prototypage rapide vs. industrialisation. Documenté dans la note d’opportunité. Confirmé (ou à confirmer) par BercyHub. |
| « L’IA va remplacer des agents » | Le Lab produit des outils pour les agents, pas à leur place. Aucun poste supprimé. Les agents gagnent du temps sur les tâches répétitives pour se concentrer sur leur cœur de métier. |
| « C’est un homme seul, si il part tout s’arrête » | Code publié sur GitHub, documenté, testé, en architecture standard. N’importe quel développeur peut reprendre. Le plan prévoit la montée à 2-3 agents. |
| « Les données vont fuiter via l’IA » | Le Lab ne traite aucune donnée sensible. Les prompts contiennent des spécifications techniques, pas des données. Le code produit est public. |
| « Ça va devenir inmaintenable » | 1 800+ tests automatisés, CI/CD, typage strict, conformité DSFR. Et le principe Grist pour ExcelExit : si l’interface meurt, les données vivent. |
| « On n’a pas le budget » | ~1 000 €/mois de licences IA. L’infra Grist est gratuite (DINUM). Le coût principal est le temps-agent, déjà alloué. ROI démontré à 88 % d’économie. |
| « Pourquoi le SNUM fait de l’innovation ? » | Parce que l’innovation web est dans la mission MIWEB. Et parce qu’un Lab interne coûte 100 fois moins qu’un projet classique pour des résultats comparables. |
