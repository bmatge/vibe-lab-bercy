La proposition

Les 5 règles cardinales

1 Prototype, pas produit

Le Vibe Lab produit des prototypes fonctionnels, jamais des applications de production. L'industrialisation passe par les filières classiques.

2 Besoin métier d'abord

Chaque projet répond à un irritant métier identifié. Pas de technologie pour la technologie.

3 Transparence totale

Code ouvert, documentation partagée, résultats publiés. Tout est auditable et reproductible.

4 Réversibilité garantie

Aucune dépendance créée. Si le Lab s'arrête, les outils existants continuent de fonctionner.

5 Données non sensibles

Le Lab ne traite que des données non confidentielles. Les données sensibles passent par BercyHub.

Le modèle éclaireur : 3 temps

Le Vibe Lab fonctionne comme un éclaireur : il explore rapidement, valide avec les utilisateurs, puis transmet à la filière pour industrialisation.

1. Explorer

Prototypage rapide en vibe coding (4h). L'agent métier exprime son besoin, l'IA génère le code. Premier livrable tangible immédiat.

2. Valider

Test utilisateur, scoring via la grille d'évaluation (/50), itérations. Si le score est suffisant, le projet passe au comité de priorisation.

3. Industrialiser

Transfert vers la filière SI classique ou vers BercyHub pour les données sensibles. Le prototype devient cahier des charges vivant.

Articulation avec BercyHub

BercyHub est le hub souverain d'IA du ministère pour les données sensibles. Le Vibe Lab et BercyHub sont complémentaires : le Lab explore avec des données non sensibles via des outils commerciaux (Claude, Copilot), puis transfère vers BercyHub quand le projet nécessite des données protégées. Une synchronisation trimestrielle assure la cohérence des deux démarches.
